## ⏳ Лишилось (потребує архітектурного рішення)

### 6 + 7. Єдиний підхід до перекладу, повний словник

**Зараз:** дві несумісні схеми одночасно —
- ключі словника (`data-translate="key"`): головна, about, contacts, publications, reports;
- інлайн (`data-translate-en="…"`): news, events, donate, join.

Через це `uk.json` тримається «наполовину»: система працює українською лише тому,
що вихідний HTML вже українською і спрацьовує фолбек `original`. Будь-яке
розходження HTML ↔ словник дає баг (саме звідси були пункти 2–4).

**Що зробити (вибрати ОДИН підхід):**

**Варіант A — повні словники (рекомендовано для SSR-i18n нижче).**
1. Винести весь текст у `uk.json` + `en.json` (зараз ~137 ключів у розмітці проти 52 в `uk.json`).
2. Прибрати інлайн `data-translate-en` — лишити тільки `data-translate="key"`.
3. Текст у розмітці лишити як SSR-дефолт, що збігається з `uk.json` (захист від FOUC).
4. Додати CI-перевірку «паритету ключів»: скрипт, що порівнює набір ключів у
   `uk.json` і `en.json` і падає на розбіжності (див. нижче приклад).

**Варіант B — лишити лише інлайн `data-translate-{lang}`** на всіх сторінках,
видалити `*.json`. Простіше, але не масштабується на 3+ мови і дублює текст у кожному файлі.

> Рекомендація: **Варіант A**, бо він — передумова для пунктів 9–10.

### 9. `<title>` / meta / `<html lang>` не перекладаються

**Зараз:** заголовок, `og:title`, `description`, `og:locale=uk_UA`, `<html lang>`
рендеряться на сервері українською (`Base.astro`) і ніколи не оновлюються при
переключенні на EN. Погано для SEO та шарингу.

**Що зробити:**
- Передавати в `Base.astro` локалізовані `title`/`description` (об'єкт `{ uk, en }` від сторінки).
- При клієнтському переключенні мови у `app.js` оновлювати `document.title`,
  `<meta name="description">`, `og:title/description/locale`, `document.documentElement.lang`.
- Краще — рендерити правильні meta одразу на сервері для кожного мовного URL (див. п.10),
  тоді клієнтське оновлення стає необов'язковим.

### 10. Тільки клієнтський переклад → FOUC + немає окремих URL для EN

**Зараз:** при збереженому `en` сторінка спершу малюється українською, потім
мигає на англійську (FOUC). Англійської версії як окремого URL не існує — пошуковики
не індексують EN-контент, немає `hreflang`.

**Що зробити — перейти на SSR/SSG-локалізацію за URL:**

1. **Маршрути за локаллю.** Astro i18n routing:
   ```js
   // astro.config.mjs
   import { defineConfig } from "astro/config";
   export default defineConfig({
     site: "https://avku.org",
     base: "/",
     i18n: {
       defaultLocale: "uk",
       locales: ["uk", "en"],
       routing: { prefixDefaultLocale: false }, // uk: /about, en: /en/about
     },
   });
   ```
2. **Структура сторінок:** `src/pages/[...]` для `uk` + `src/pages/en/[...]` (або
   динамічний `getStaticPaths` по локалях), увесь текст — через `t(key, locale)`.
3. **`hreflang` у `<head>`** (`Base.astro`):
   ```html
   <link rel="alternate" hreflang="uk" href="https://avku.org/about" />
   <link rel="alternate" hreflang="en" href="https://avku.org/en/about" />
   <link rel="alternate" hreflang="x-default" href="https://avku.org/about" />
   ```
4. **Коректні per-locale meta:** `<html lang>`, `og:locale` (`uk_UA` / `en_US`),
   `og:url`, `canonical` — рахувати з поточної локалі.
5. **Перемикач мови** перетворити з кнопок `data-lang` на посилання на той самий
   шлях в іншій локалі (зберігати вибір у cookie для редіректу з кореня).
6. **Прибрати FOUC:** після SSR-локалізації клієнтський `app.js` стає легасі —
   або видалити, або лишити лише для збереження бажаної мови та редіректу.
7. **`sitemap.xml`** — додати обидві локалі з `hreflang`-анотаціями (`@astrojs/sitemap` з `i18n`).

## Приклад CI-перевірки паритету ключів (для п.6–7)

```js
// scripts/check-i18n.mjs
import fs from "node:fs";
const uk = JSON.parse(fs.readFileSync("public/lang/uk.json", "utf8"));
const en = JSON.parse(fs.readFileSync("public/lang/en.json", "utf8"));
const a = new Set(Object.keys(uk)), b = new Set(Object.keys(en));
const onlyUk = [...a].filter((k) => !b.has(k));
const onlyEn = [...b].filter((k) => !a.has(k));
if (onlyUk.length || onlyEn.length) {
  console.error("Key mismatch:", { onlyUk, onlyEn });
  process.exit(1);
}
console.log("i18n keys in sync:", a.size);
```
